
Ưu điểm của Docker:
- Tính di động: Docker cho phép bạn đóng gói ứng dụng và tất cả các phụ thuộc của nó vào một container, giúp ứng dụng có thể chạy trên bất kỳ môi trường nào hỗ trợ Docker mà không cần phải lo lắng về sự khác biệt giữa các hệ điều hành hoặc cấu hình phần cứng.
- Hiệu suất: Docker sử dụng tài nguyên hệ thống một cách hiệu quả hơn so với các máy ảo truyền thống, vì nó chia sẻ kernel của hệ điều hành chủ và chỉ tạo ra các container nhẹ hơn.
- Dễ dàng triển khai: Docker giúp đơn giản hóa quá trình triển khai ứng dụng bằng cách cho phép bạn tạo ra các container có thể được triển khai nhanh chóng và dễ dàng trên các môi trường khác nhau, từ máy tính cá nhân đến các máy chủ đám mây.
- Quản lý phiên bản: Docker cho phép bạn quản lý các phiên bản của ứng dụng và các phụ thuộc của nó một cách dễ dàng, giúp bạn có thể quay lại các phiên bản trước đó nếu cần thiết.
- Hỗ trợ cộng đồng: Docker có một cộng đồng lớn và sôi động, cung cấp nhiều tài nguyên, công cụ và hỗ trợ cho người dùng, giúp bạn có thể dễ dàng tìm kiếm giải pháp cho các vấn đề mà bạn gặp phải khi sử dụng Docker.
- Tích hợp với CI/CD: Docker tích hợp tốt với các công cụ CI/CD, giúp bạn có thể tự động hóa quá trình xây dựng, kiểm thử và triển khai ứng dụng một cách hiệu quả hơn.
- Bảo mật: Docker cung cấp các tính năng bảo mật như cách ly container, kiểm soát truy cập và quản lý quyền hạn, giúp bảo vệ ứng dụng của bạn khỏi các mối đe dọa bảo mật.
- Khả năng mở rộng: Docker cho phép bạn dễ dàng mở rộng ứng dụng của mình bằng cách tạo ra nhiều container và quản lý chúng một cách hiệu quả, giúp bạn có thể đáp ứng nhu cầu tăng trưởng của ứng dụng một cách linh hoạt.
- Dễ dàng quản lý: Docker cung cấp các công cụ quản lý container mạnh mẽ, giúp bạn có thể dễ dàng theo dõi, kiểm soát và quản lý các container của mình một cách hiệu quả.
- Hỗ trợ đa nền tảng: Docker hỗ trợ nhiều nền tảng khác nhau, bao gồm Windows, macOS và Linux, giúp bạn có thể sử dụng Docker trên bất kỳ hệ điều hành nào mà bạn ưa thích.
- Tính linh hoạt: Docker cho phép bạn dễ dàng tùy chỉnh và cấu hình các container của mình để phù hợp với nhu cầu cụ thể của ứng dụng, giúp bạn có thể tối ưu hóa hiệu suất và tài nguyên của ứng dụng một cách hiệu quả hơn.
- Tính nhất quán: Docker giúp đảm bảo rằng ứng dụng của bạn sẽ chạy một cách nhất quán trên mọi môi trường, từ phát triển đến sản xuất, giúp giảm thiểu các vấn đề liên quan đến sự khác biệt giữa các môi trường.
- Hỗ trợ microservices: Docker là một công cụ lý tưởng cho kiến trúc microservices, cho phép bạn dễ dàng triển khai và quản lý các dịch vụ nhỏ hơn một cách độc lập, giúp tăng tính linh hoạt và khả năng mở rộng của ứng dụng.
- Tích hợp với các công cụ khác: Docker tích hợp tốt với nhiều công cụ khác nhau, bao gồm các công cụ quản lý cấu hình, các công cụ giám sát và các công cụ quản lý container khác, giúp bạn có thể xây dựng một hệ sinh thái công cụ mạnh mẽ để quản lý ứng dụng của mình một cách hiệu quả hơn.
- Hỗ trợ DevOps: Docker là một phần quan trọng của quy trình DevOps, giúp tăng cường sự hợp tác giữa các nhóm phát triển và vận hành, giúp bạn có thể triển khai ứng dụng một cách nhanh chóng và hiệu quả hơn.
- Tính năng đa dạng: Docker cung cấp nhiều tính năng mạnh mẽ như Docker Compose, Docker Swarm và Kubernetes, giúp bạn có thể quản lý và triển khai các ứng dụng phức tạp một cách dễ dàng hơn.
- Tính năng tự động hóa: Docker cho phép bạn tự động hóa quá trình xây dựng, kiểm thử và triển khai ứng dụng, giúp bạn có thể tiết kiệm thời gian và công sức trong việc quản lý ứng dụng của mình.
- Hỗ trợ cộng đồng: Docker có một cộng đồng lớn và sôi động, cung cấp nhiều tài nguyên, công cụ và hỗ trợ cho người dùng, giúp bạn có thể dễ dàng tìm kiếm giải pháp cho các vấn đề mà bạn gặp phải khi sử dụng Docker.
- Tính năng bảo mật: Docker cung cấp các tính năng bảo mật như cách ly container, kiểm soát truy cập và quản lý quyền hạn, giúp bảo vệ ứng dụng của bạn khỏi các mối đe dọa bảo mật.
- Tính năng mở rộng: Docker cho phép bạn dễ dàng mở rộng ứng dụng của mình bằng cách tạo ra nhiều container và quản lý chúng một cách hiệu quả, giúp bạn có thể đáp ứng nhu cầu tăng trưởng của ứng dụng một cách linh hoạt.

# CÁC KHÁI NIỆM:

Khi làm việc với Docker, có một số khái niệm cơ bản mà bạn cần hiểu để có thể sử dụng Docker một cách hiệu quả. Dưới đây là một số khái niệm quan trọng liên quan đến Docker:



## IMAGES:
Image: Là một bản sao của một hệ điều hành hoặc ứng dụng được đóng gói dưới dạng một file có thể chạy trên Docker. Image chứa tất cả các thành phần cần thiết để chạy ứng dụng, bao gồm mã nguồn, thư viện, cấu hình và các phụ thuộc khác.

## CONTAINERS:
Container: Là một instance đang chạy của một image. Container là một môi trường ảo hóa nhẹ, cho phép bạn chạy ứng dụng một cách độc lập và cách ly với hệ điều hành chủ. Container có thể được tạo, bắt đầu, dừng và xóa một cách dễ dàng.

## DOCKERFILE:
Dockerfile: Là một file văn bản chứa các lệnh để xây dựng một image Docker tùy chỉnh. Dockerfile cho phép bạn định nghĩa cách thức xây dựng image, bao gồm việc cài đặt các gói phần mềm, sao chép các file vào image và cấu hình môi trường cho ứng dụng của bạn.

## DOCKER HUB:
Docker Hub: Là một dịch vụ lưu trữ và chia sẻ image Docker trực tuyến. Docker Hub cho phép bạn tìm kiếm, tải về và chia sẻ các image Docker với cộng đồng. Bạn có thể sử dụng Docker Hub để tìm kiếm các image có sẵn cho các ứng dụng phổ biến hoặc để lưu trữ và chia sẻ các image Docker tùy chỉnh của riêng bạn.

## DOCKER COMPOSE:
Docker Compose: Là một công cụ để định nghĩa và chạy các ứng dụng đa container. Docker Compose cho phép bạn sử dụng một file YAML để định nghĩa các dịch vụ, mạng và volume cần thiết cho ứng dụng của bạn, và sau đó sử dụng một lệnh duy nhất để khởi động tất cả các container của ứng dụng một cách dễ dàng.

## DOCKER SWARM:
Docker Swarm: Là một công cụ để quản lý và triển khai các ứng dụng Docker trên nhiều máy chủ. Docker Swarm cho phép bạn tạo ra một cluster Docker Swarm, nơi bạn có thể triển khai và quản lý các ứng dụng phức tạp hơn trên nhiều máy chủ một cách dễ dàng.

## KUBERNETES:
Kubernetes: Là một hệ thống mã nguồn mở để tự động hóa việc triển khai, mở rộng và quản lý các ứng dụng container. Kubernetes cung cấp một nền tảng mạnh mẽ để quản lý các ứng dụng phức tạp trên nhiều máy chủ, bao gồm các tính năng như cân bằng tải, tự động mở rộng và phục hồi sau sự cố. Kubernetes là một công cụ phổ biến để triển khai và quản lý các ứng dụng container trong môi trường sản xuất.

## DOCKER DASHBOARD:
Docker Dashboard: Là một giao diện người dùng đồ họa để quản lý các container Docker của bạn. Docker Dashboard cho phép bạn theo dõi hiệu suất, trạng thái và các thông tin khác về các container của bạn một cách dễ dàng, giúp bạn có thể quản lý và kiểm soát các container của mình một cách hiệu quả hơn.

## PORTAINER:
Portainer: Là một công cụ quản lý container Docker dựa trên web. Portainer cung cấp một giao diện người dùng đơn giản và trực quan để quản lý các container Docker của bạn, bao gồm các tính năng như theo dõi hiệu suất, quản lý mạng và volume, và quản lý người dùng và quyền hạn. Portainer giúp bạn có thể quản lý các container Docker của mình một cách dễ dàng và hiệu quả hơn.

## CADVISOR:
cAdvisor: Là một công cụ giám sát container Docker mã nguồn mở. cAdvisor cung cấp các thông tin chi tiết về hiệu suất và trạng thái của các container Docker của bạn, bao gồm các thông tin về CPU, bộ nhớ, mạng và đĩa. cAdvisor giúp bạn có thể theo dõi và phân tích hiệu suất của các container Docker của mình một cách dễ dàng và hiệu quả hơn.

## BẢO MẬT DOCKER:
Docker cung cấp các tính năng bảo mật như cách ly container, kiểm soát truy cập và quản lý quyền hạn để bảo vệ ứng dụng của bạn khỏi các mối đe dọa bảo mật. Bạn có thể sử dụng các tính năng này để đảm bảo rằng ứng dụng của bạn được bảo vệ một cách hiệu quả khi chạy trong container Docker. Ngoài ra, bạn cũng có thể áp dụng các biện pháp bảo mật bổ sung cho container của mình, như sử dụng các image Docker được xây dựng từ các nguồn đáng tin cậy, sử dụng các công cụ quét bảo mật để kiểm tra các image Docker của bạn và áp dụng các chính sách bảo mật cho container của bạn để giảm thiểu các rủi ro bảo mật.

## TỐI ƯU HÓA HIỆU SUẤT:
Docker cung cấp các kỹ thuật như caching, multi-stage builds và resource limits để tối ưu hóa hiệu suất của ứng dụng khi chạy trong container Docker. Caching cho phép bạn lưu trữ các lớp trung gian của image Docker để giảm thời gian xây dựng lại image khi có sự thay đổi nhỏ trong Dockerfile. Multi-stage builds cho phép bạn tách quá trình xây dựng image thành nhiều giai đoạn, giúp giảm kích thước của image cuối cùng và cải thiện hiệu suất. Resource limits cho phép bạn giới hạn tài nguyên mà container của bạn có thể sử dụng, giúp đảm bảo rằng ứng dụng của bạn chạy một cách hiệu quả và không gây ảnh hưởng đến hiệu suất của hệ thống chủ.

## TÍCH HỢP VỚI QUY TRÌNH PHÁT TRIỂN VÀ TRIỂN KHAI:
Docker có thể được tích hợp vào quy trình phát triển và triển khai ứng dụng của bạn để tăng cường sự linh hoạt và hiệu quả của quy trình phát triển và triển khai ứng dụng của bạn. Bạn có thể sử dụng Docker trong môi trường phát triển để tạo ra các container phát triển độc lập, giúp bạn có thể làm việc trên ứng dụng của mình một cách hiệu quả hơn. Bạn cũng có thể sử dụng Docker trong môi trường kiểm thử để tạo ra các container kiểm thử độc lập, giúp bạn có thể kiểm thử ứng dụng của mình một cách hiệu quả hơn. Cuối cùng, bạn có thể sử dụng Docker trong môi trường sản xuất để triển khai ứng dụng của mình một cách nhanh chóng và hiệu quả hơn, giúp bạn có thể đáp ứng nhu cầu của người dùng một cách linh hoạt hơn.

## CÔNG CỤ VÀ DỊCH VỤ LIÊN QUAN ĐẾN DOCKER:
Docker Hub, Docker Registry, Docker Compose, Docker Swarm, Kubernetes và các công cụ giám sát và quản lý container khác là các công cụ và dịch vụ liên quan đến Docker mà bạn có thể sử dụng để xây dựng một hệ sinh thái công cụ mạnh mẽ để quản lý ứng dụng của bạn một cách hiệu quả hơn. Docker Hub và Docker Registry cho phép bạn lưu trữ và chia sẻ các image Docker của bạn, trong khi Docker Compose, Docker Swarm và Kubernetes cung cấp các công cụ để quản lý và triển khai các ứng dụng Docker phức tạp hơn. Các công cụ giám sát và quản lý container như Docker Dashboard, Portainer và cAdvisor giúp bạn theo dõi và quản lý các container Docker của mình một cách hiệu quả hơn, giúp bạn có thể đảm bảo rằng ứng dụng của bạn chạy một cách hiệu quả và ổn định trong môi trường Docker.

Trong phát triển phần mềm, Docker giúp các nhà phát triển tạo ra các môi trường phát triển độc lập và dễ dàng chia sẻ các môi trường này với các thành viên khác trong nhóm. Trong triển khai ứng dụng, Docker giúp các nhà phát triển triển khai ứng dụng một cách nhanh chóng và hiệu quả hơn, giúp đáp ứng nhu cầu của người dùng một cách linh hoạt hơn. Trong quản lý hạ tầng, Docker giúp các nhà quản trị hệ thống quản lý và triển khai các ứng dụng một cách hiệu quả hơn, giúp giảm thiểu các vấn đề liên quan đến sự khác biệt giữa các môi trường và tăng cường sự linh hoạt trong việc quản lý hạ tầng của bạn. Ngoài ra, Docker cũng được sử dụng trong các lĩnh vực khác như học máy, phân tích dữ liệu và các lĩnh vực khác để giúp tăng cường hiệu suất và tính linh hoạt của các ứng dụng trong các lĩnh vực này.

## EXAMPLE 02: TẠO VÀ CHẠY MỘT CONTAINER DOCKER ĐƠN GIẢN

### ĐỀ BÀI:
Tạo một ứng dụng đơn giản (python) xuất ra "Hello, Docker!" và đóng gói nó vào một container Docker. Sau đó, chạy container đó để xem kết quả.
## CẤU TRÚC THƯ MỤC

## STEP01: TẠO ỨNG DỤNG PYTHON ĐƠN GIẢN
Tạo một file Python đơn giản có tên `docker_example_02.py` với nội dung sau:
```python
#!/usr/bin/env python3

print("Hello, Docker - Example 02!")
```
> **Giải thích:** Đây là một file Python đơn giản, khi được chạy sẽ in ra thông báo "Hello, Docker - Example 02!" trên màn hình.
## STEP02: TẠO DOCKERFILE
Tạo một file `dockerfile` trong cùng thư mục với nội dung sau:
```Dockerfile
# Sử dụng image Python chính thức làm base image
FROM python:3

# Khai báo thư mục làm việc trong container
WORKDIR /app

# Sao chép docker_example_02.py vào thư mục /app trong container
COPY docker_example_02.py .

# Chạy file Python khi container được khởi động
CMD ["python", "docker_example_02.py"]
```
> **Giải thích:** Dockerfile này sử dụng image Python chính thức làm base image, thiết lập thư mục làm việc trong container là `/app`, sao chép file `docker_example_02.py` vào thư mục làm việc trong container và chỉ định lệnh để chạy file Python khi container được khởi động.
## STEP03: XÂY DỰNG IMAGE DOCKER
Mở terminal và điều hướng đến thư mục chứa Dockerfile và file Python của bạn, sau đó chạy lệnh sau để xây dựng image Docker từ Dockerfile:
```bash
docker build -t Rom/docker-example-02 .
```
> **Giải thích:** Lệnh này sẽ xây dựng một image Docker từ Dockerfile trong thư mục hiện tại và gắn thẻ image đó với tên `Rom/docker-example-02`. Dấu chấm ở cuối lệnh chỉ định rằng Dockerfile nằm trong thư mục hiện tại.

```bash
docker images | grep docker-example-02
```
> **Giải thích:** Lệnh này sẽ liệt kê tất cả các image Docker đã có trên máy tính của bạn, để kiểm tra image `Rom/docker-example-02` đã được xây dựng thành công hay chưa.
## STEP04: CHẠY CONTAINER TỪ IMAGE DOCKER
Sau khi image đã được xây dựng thành công, chạy lệnh sau để tạo và chạy một container từ image đó:
```bash
docker run --name docker-example-02 Rom/docker-example-02
```
> **Giải thích:** Lệnh này sẽ tạo và chạy một container từ image `Rom/docker-example-02` và đặt tên cho container là `docker-example-02`. Khi container được khởi động, nó sẽ thực thi lệnh được chỉ định trong Dockerfile, đó là chạy file Python `docker_example_02.py`.
## RESULT:
Bạn sẽ thấy thông báo "Hello, Docker - Example 02!" được in ra từ container, cho biết rằng ứng dụng Python của bạn đã chạy thành công trong container Docker.

```Dockerfile
CMD ["Tham số mặc định"]
```
> **Giải thích:** Pattern này kết hợp cả hai. `ENTRYPOINT` chỉ định lệnh chính, `CMD` cung cấp tham số mặc định.
> - Nếu **có truyền args**: args sẽ ghi đè CMD
> - Nếu **không truyền args**: CMD sẽ được sử dụng

**Xây dựng image:**
```bash
cd latest-entrypoint-cmd
docker build -t Rom/docker-example-05:latest-entrypoint-cmd .
```

**Chạy container:**
```bash
# Chạy với CMD (không truyền args - dùng tham số mặc định) - ✅ Thành công
docker run --name docker-example-05-ep-cmd-latest Rom/docker-example-05:latest-entrypoint-cmd

# Chạy với args truyền vào (ghi đè CMD) - ✅ Thành công
docker run --name docker-example-05-ep-cmd-latest-args Rom/docker-example-05:latest-entrypoint-cmd "Tham số tùy chỉnh"
```

## STEP06: TEST VÀ SO SÁNH CÁC PATTERN
Chạy từng pattern và quan sát kết quả:

**Pattern CMD only:**
```bash
docker run Rom/docker-example-05:v1.0-cmd                    # ✅ Chạy được
docker run Rom/docker-example-05:v1.0-cmd "test arg"        # ❌ Lỗi
```

**Pattern ENTRYPOINT only:**
```bash
docker run Rom/docker-example-05:v2.0-entrypoint            # ✅ Chạy được
docker run Rom/docker-example-05:v2.0-entrypoint "test arg" # ✅ Thành công
```

**Pattern ENTRYPOINT + CMD:**
```bash
docker run Rom/docker-example-05:latest-entrypoint-cmd      # ✅ Chạy được (dùng CMD default)
docker run Rom/docker-example-05:latest-entrypoint-cmd "test arg"  # ✅ Thành công
```

## RESULT:
Bạn sẽ thấy rõ sự khác biệt giữa các pattern:

- **CMD only** (v1.0-cmd): Chỉ chạy được khi không truyền args. Khi truyền args, lệnh bị ghi đè hoàn toàn → Lỗi.
- **ENTRYPOINT only** (v2.0-entrypoint): Luôn chạy lệnh entrypoint, args được truyền vào. Có thể truyền args.
- **ENTRYPOINT + CMD** (latest-entrypoint-cmd): Linh hoạt - có args dùng args, không có args dùng CMD default.

Mỗi pattern là một image **độc lập** với tag khác nhau, nên bạn có thể test và so sánh hành vi dễ dàng.
