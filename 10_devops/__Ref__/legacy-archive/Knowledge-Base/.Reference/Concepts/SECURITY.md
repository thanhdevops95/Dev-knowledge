# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please follow these steps:

1. **Do NOT** create a public GitHub issue
2. Email the maintainer directly at: [Create a private security advisory](https://github.com/thanhlehoang0107/DevOps-Mastery/security/advisories/new)
3. Include as much information as possible:
   - Type of vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- Initial response: Within 48 hours
- Status update: Within 7 days
- Fix timeline: Depends on severity

## Scope

This security policy applies to:

- Main repository content
- Example code and scripts
- Configuration templates

Note: This is an educational repository. Most content is documentation rather than executable production code.

Thank you for helping keep DevOps Mastery secure!
