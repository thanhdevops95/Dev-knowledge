## 📊 Cấp độ bài viết

Mỗi bài được gắn badge ở đầu để bạn biết mình cần chuẩn bị gì:

| Badge | Cấp độ | Ý nghĩa |
|---|---|---|
| 🟢 `[BEGINNER]` | Người mới | Không cần kiến thức trước — bắt đầu từ đây |
| 🟡 `[INTERMEDIATE]` | Trung cấp | Cần đã nắm basics (ghi rõ prerequisite) |
| 🔴 `[ADVANCED]` | Nâng cao | Yêu cầu kinh nghiệm thực tế |
| ⭐ `[MUST-KNOW]` | Bắt buộc biết | Kiến thức cốt lõi — bỏ qua sẽ thiếu nền tảng |

## 📁 Cách tổ chức bài viết

Mỗi file được đặt tên theo format: **`[STT]-[tên-kebab-case]-[suffix].md`**

Suffix cho biết **loại bài viết** — bạn có thể chọn đọc theo nhu cầu:

| Nhu cầu | Đọc bài suffix | Ví dụ |
|---|---|---|
| Hiểu khái niệm nền tảng | `-fundamentals` | `oop-fundamentals`, `tcp-ip-fundamentals` |
| Bắt đầu học tool/ngôn ngữ | `-basics` | `python-basics`, `docker-basics` |
| Đi sâu nâng cao | `-advanced` | `python-advanced`, `k8s-networking-advanced` |
| Hiểu cách hoạt động bên trong | `-deep-dive` | `tcp-deep-dive`, `database-deep-dive` |
| Học best practices | `-practices` | `tdd-practices`, `code-review-practices` |
| Xem code mẫu thực tế | `-examples` | `api-design-examples`, `git-examples` |
| Cài đặt / cấu hình | `-setup` | `python-packaging-setup` |
| Học design patterns | `-patterns` | `gof-patterns`, `caching-strategies-patterns` |
| Tra cứu nhanh | `-cheatsheet` | `git-cheatsheet`, `sql-cheatsheet` |
| So sánh công nghệ | `-compare` | `frontend-frameworks-compare` |
| Ôn thi chứng chỉ | `-cert` | `aws-saa-cert`, `cka-cert` |

## ✍️ Đóng góp

Mọi đóng góp đều được chào đón! Đọc **[CONTRIBUTING.md](./CONTRIBUTING.md)** — tài liệu duy nhất chứa:

- Quy tắc đặt tên, phân loại bài viết (§1-§3)
- Bố cục nội dung bắt buộc cho từng loại bài (§4-§6)
- Template, phong cách viết, checklist (§7-§12)
- Template copy-paste sẵn dùng (§13)
- **Outline chi tiết cho toàn bộ 421 bài** (§14-§37)

```ini
Quy trình đóng góp:
1. Mở CONTRIBUTING.md → tìm bài muốn viết (Phần 2, §14-§36)
2. Đọc outline + quy tắc suffix (Phần 1, §2-§6)
3. Tạo file theo naming convention + template
4. Kiểm tra checklist (§11) → Submit
```
