> 🎯 *Quản lý thời gian + năng suất cá nhân cho knowledge worker: vì sao dev cần cách riêng (context switch đắt, energy > time), ưu tiên công việc, deep work + time blocking, hệ thống task/lập kế hoạch (GTD), và giữ nhịp độ bền vững tránh quá tải. Nhiều framework + template + checklist, không code.*

---

## 🎯 Mục tiêu tổng

Sau khi đi qua chủ đề này, bạn sẽ:
- [x] Hiểu vì sao knowledge work cần quản lý thời gian khác (context switch, maker vs manager schedule, energy management)
- [x] **Ưu tiên** bằng Eisenhower / 80/20 / eat-the-frog / MoSCoW, biết nói "không"
- [x] Bảo vệ **deep work** bằng **time blocking** + Pomodoro, giảm context switching
- [x] Dùng **hệ thống task** (GTD nhẹ) + daily/weekly planning, ước lượng có buffer (planning fallacy)
- [x] Giữ **nhịp độ bền vững**, chống procrastination + overcommitment + quá tải
