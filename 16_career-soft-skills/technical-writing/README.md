> 🎯 *Technical writing — siêu năng lực bị đánh giá thấp của dev. Học viết README/tài liệu dự án, design doc/RFC + ADR, tài liệu API & code (docstring/changelog), và truyền đạt bằng sơ đồ (mermaid, C4). Nhiều template + ví dụ before/after, áp dụng được ngay.*

---

## 🎯 Mục tiêu tổng

Sau khi đi qua chủ đề này, bạn sẽ:
- [x] Hiểu vì sao tài liệu tốt = scale kiến thức; mô hình **Diátaxis** (tutorial/how-to/reference/explanation)
- [x] Viết **README + tài liệu dự án** theo docs-as-code, quick-start chạy được trong 5 phút
- [x] Viết **design doc/RFC** + **ADR** để suy nghĩ rõ + lấy buy-in + lưu vết quyết định
- [x] Làm **tài liệu API** (OpenAPI) + comment/docstring "giải thích WHY" + changelog chuẩn
- [x] Truyền đạt bằng **sơ đồ** (mermaid, C4 model), biết khi nào & vẽ sơ đồ tốt
